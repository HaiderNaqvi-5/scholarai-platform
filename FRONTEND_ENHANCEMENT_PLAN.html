02 — Current Frontend Problems

What's broken or missing right now

No Pakistan identity

Landing says "For grad school applicants." Brand is "GrantPath" on landing but "ScholarAI" everywhere else. No Pakistani student feels this was made for them.

Onboarding misses Pakistani fields

Current form asks generic GPA/citizenship — no university name autocomplete, no CGPA scale context, no intake target, no financial context. Students can't complete a meaningful profile.

5 features are placeholder stubs

Scholarship Finder, Application Tracker, SOP Builder, Visa Interview, and Upgrade Page all show "PlaceholderRoute." The product is incomplete for the demo path.

Dashboard is opaque

"For you" and "Discover" labels tell the student nothing about what actions to take. No clear next step, no urgency, no features surfaced.

No freemium model in UI

No UpgradeWall, no blur effect, no pricing page, no plan gates. The freemium model is in the PRD but not implemented anywhere in the frontend.

Mobile not tested

Pakistani students are mobile-first. No 375px testing, sidebar likely broken on mobile, kanban not responsive. This is a critical gap.

03 — Design Principles for This Rebuild

Non-negotiable UX rules

These override any default behavior. GoScholar wins on emotion — we win on depth + trust + Pakistan specificity.

⚡

Zero AI theatre

No fake streaming, no loading spinners on REST data. Scholarship cards render once. Skeletons show once per route. Instant perceived performance.

🎯

One CTA at a time

Every screen has exactly one primary action. Landing → "Find My Scholarships." Feed → "View Match." Onboarding → "Continue." No decision paralysis.

🇵🇰

Pakistan context everywhere

PKR pricing, Pakistani university names, CGPA/HEC context, visa fear language. Every screen should feel built for a student from Lahore or Karachi.

📱

Mobile-first, WhatsApp-aware

375px is the primary canvas. Share-to-WhatsApp on scholarship cards. Bottom navigation on mobile. Tap targets ≥44px on every action.

🔒

Show locked, never hide

Free tier sees blurred results with counts: "9 more matches including 2 fully funded." UpgradeWall shows specific context from the API — never generic "unlock this."

✅

Validated ≠ AI generated

Scholarship facts (deadlines, amounts) always in validated green stripe. AI-generated text (SOP draft, explanations) always in generated blue stripe + Sparkles icon.

🏃

Frictionless onboarding

Under 3 minutes. Smart defaults for common Pakistani universities. localStorage autosave every keystroke. No required fields that block progress except degree + countries.

🎨

No emoji as UI

Lucide SVG icons only in UI chrome. Emoji only in content (country flags in scholarship names, the PRD uses them — keep in data, not chrome).

📊

Urgency is real, not fake

"Chevening closes in 47 days" is a real deadline countdown. Scholarship cards show red border when <30 days. Never manufactured false urgency.

04 — Unique Differentiators vs GoScholar

What we have that they don't — and must make visible

Visa Interview Simulator — First in Market

GoScholar's "Visa Preparation" is listed as "Coming Soon." We have 70 seeded questions for UK, USA, Canada, Germany with AI rubric evaluation. This is the headline feature — put it front and center on landing and dashboard.

Pakistani CGPA Conversion Context

"Your 3.6 CGPA from NUST is equivalent to a 3.5 US GPA" — no competitor shows this. Surface in every scholarship match card. Makes our eligibility filtering feel more honest and expert.

SOP with Pakistani Academic Context

Understands HEC degrees, Pakistani university prestige, ties-to-Pakistan framing for government scholarships, visa officer reading lens. GoScholar's AI assistant is generic.

HEC Attestation Awareness

Every application tracker card has HEC Degree Attestation in the checklist — labeled "required for UK/Germany." This is a Pakistani-specific pain point no other platform surfaces.

PKR-First Freemium Pricing

"Less than one consultant meeting — PKR 2,499/month." Positioned specifically against the consultant market in Pakistan. Conversion copy that resonates culturally, not just functionally.

Multi-Channel Alert System

Elite: Email + SMS + WhatsApp alerts for deadlines. WhatsApp is the communication layer in Pakistan. GoScholar has no WhatsApp integration. This is a real retention moat.

05 — Onboarding Flow Redesign

Pakistan-specific 4-step onboarding (under 3 minutes)

Replace the current generic 4-step form. Every field must feel relevant to a Pakistani MS applicant. Smart defaults, no walls, autosave on every change.

Step 1

About You

  • Full name
  • City (Karachi, Lahore, Islamabad…)
  • Pakistani university (autocomplete: NUST, LUMS, FAST, UET, IBA, QAU, COMSATS…)
  • Degree subject (CS, EE, Data Science…)
  • Graduation year
~45 seconds
Step 2

Academic Record

  • CGPA value
  • CGPA scale (4.0 standard / 4.0 HEC)
  • HEC degree level (Bachelor / Master / MPhil)
  • Research publications? (yes/no + count)
  • Show US GPA equivalent live
~45 seconds
Step 3

Test Scores

  • IELTS score (or "not taken yet")
  • TOEFL score (optional)
  • GRE Quant / Verbal (optional)
  • GRE waiver needed? (toggle)
  • Clear "all optional" messaging
~30 seconds
Step 4

Your Goal

  • Target degree (MS / PhD / MBA)
  • Target countries (multi-select flags)
  • Target fields (CS, DS/AI, EE…)
  • Funding: Fully funded only / Partial OK
  • Intake target (Sep 2025 / Jan 2026…)
~45 seconds
Step 5

Financial Context

  • Can afford application fees?
  • Family has funds for bank statement?
  • Brief — 3 toggles only
  • Skip option visible
  • → Redirect to Dashboard
~20 seconds

Critical UX decisions for onboarding

1. Show progress bar at top with steps + "~3 min" estimate.
2. Show CGPA US-equivalent inline as student types ("Your 3.6 = ~3.5 US GPA") — builds trust immediately.
3. Pakistani university field: searchable dropdown with 20 top universities pre-loaded, free-text fallback.
4. Step 4 country select: show country flags as large tappable cards (not a dropdown) — mobile-friendly + satisfying.
5. "Continue" should always be enabled — no field blocks progress except target_degree + target_countries on Step 4.
6. On the final step, CTA changes to "Find My Scholarships →" not "Save."

Visa Interview Simulator
Route: /dashboard/interview — UK/USA/Canada/Germany. First-in-market feature.
Headline differentiator Study + Exam modes Depends on: PRD §8 backend
4dev-days
What to build
  • Setup screen: Country cards (UK flag, USA, Canada, Germany) + mode toggle (Study/Exam) + "Start Session" CTA
  • Interview screen: Progress bar "Q 3 of 10" + timer + category badge (Finances, Ties, Motivation) + question in large text + textarea + Submit Answer
  • Feedback panel (study mode): Score meters (Clarity/Confidence/Relevance) + red flag warnings in danger-red + "What was good" in validated + "What to add" in caution + "Ideal answer" in generated + "Next Question →"
  • Exam mode: No feedback between questions — shows next question immediately after submit
  • Freemium gate: After Q3 for free tier — show partial summary + "You got 2 red flags. Finish for PKR 2,499/mo." UpgradeWall
  • Session summary: Radar chart (Clarity/Confidence/Relevance/Red Flag Avoidance) + Q-by-Q table + improvements list + Practice Again / Download Summary
  • Elite badge: "Download Transcript" button on summary for Elite users
Key UX decisions
  • Textarea: large, no character limit shown. Placeholder: "Type your answer as you would speak it. Be natural, not rehearsed."
  • Submit button: disabled when textarea empty. Shows "Evaluating..." state on click — not a spinner, just the button text changes.
  • Red flag display: each flag in a danger-soft pill. Never shown in exam mode until summary.
  • Radar chart: use a lightweight SVG radar (no Chart.js overhead) — 4 axes. Animate in on mount.
  • Timer: only visible in exam mode. Counts UP (not down) — shows session duration, not a countdown pressure.
  • Mobile: full-screen question + textarea. Feedback panel slides up from bottom as a sheet.
  • Category badge: makes student aware of what the question is testing. Rotates per question.
Acceptance gates

  • Study mode shows feedback after every answer
  • Exam mode suppresses feedback until summary
  • Red flags detected for bad answer ("I plan to stay in UK")
  • Free tier UpgradeWall shows after Q3 with red flag count
  • Summary radar chart renders correctly
