# Security Policy

Thank you for taking the time to report security issues. I take the security of the ScholarAI Platform seriously.

## Supported Versions

Currently, only the latest version of the ScholarAI Platform on the `main` branch is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| < 1.0.0 | :x:                |

*Note: As this project is in active development, older branches or pre-release versions might not receive security patches.*

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues, as this can expose the project to exploitation before a fix is ready.

If you believe you have found a security vulnerability in the ScholarAI Platform, please report it privately. You can do this by:

1. **Email:** Send a detailed report to haidernaqvi7989@gmail.com.
2. **GitHub Private Vulnerability Reporting:** If enabled on this repository, navigate to the "Security" tab, click on "Advisories", and select "Report a vulnerability".

### What to include in your report:

* A clear description of the vulnerability and its potential impact.
* Detailed steps to reproduce the issue.
* Any relevant code snippets, screenshots, or logs.
* Possible mitigations or suggestions for a fix (optional, but highly appreciated).

### Response Timeline:

I will strive to acknowledge your report within 48 hours and will keep you updated on the progress of the fix. Once the issue is resolved, a security advisory may be published to notify other users and contributors.

Thank you for helping keep the ScholarAI Platform secure!
